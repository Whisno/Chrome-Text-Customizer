About regular expressions

A regular expression (abbreviated regexp) is a sequence of characters that forms a search pattern. It is a very powerful tool (altough sometimes a little tricky) to logically define how to match series of characters.

If you want to learn how to use regexp, you can go through this tutorial. If you just want a quick syntax reference, here is a cheat sheet. And finally, you can live test your regular expressions with this tool.

Note : because of the nature of wep pages, the ^ and $ boundaries (used to match beginning and end of a string / line) will not work as expected. They will match beginning and end of text nodes, which might not look like logical text blocks.

About replacement functions

The replacement function will be executed for each match. This function receives two arguments : 'node', the DOM node in which there was a match, and 'match', the result array as returned by RegExp.prototype.exec().

So match[0] is the matched strings, match.index is the start position of the matched string in the text node's content and match[n] is the nth parenthesized substring matche (if defined).

For example, here is a function that wraps the match in a 'em' tag.

var wrap = document.createElement('em'); node.splitText(match.index+match[0].length); wrap.appendChild(node.splitText(match.index)); node.parentNode.insertBefore(wrap, node.nextSibling);
